# プロジェクト概要

## 目的

1. Next.jsフロントエンド開発のサンドボックス
2. 個人オウンドメディアとしての運用
3. AI依存からの脱却と純粋なプログラミングの実践

**重要**: サンドボックスとしての性質上、実用性よりも学習目的の実装が含まれる場合があります。

## 技術スタック

- Next.js + TypeScript
- InstantDB（データベース）
- Vercel（ホスティング）

## レビュー優先度

### 必須（Critical）

- セキュリティ上の問題
- 適切なエラーハンドリング
- 重大なパフォーマンス問題

### 高（High）

- アーキテクチャ構成と拡張性
- TypeScriptの型安全性
- 可読性・保守性

### 中（Medium）

- 関数型プログラミングの実践
- テスト可能性（Storybook/Jest対応）
- データモデル設計

### 参考（Optional）

- より良い実装パターンの提案
- 最新のベストプラクティスの紹介

## InstantDB固有の注意事項

- AGENTS.mdを参照してスキーマ設計をレビュー
- リレーショナルDBとの違いを考慮した設計指針
- InstantDBの制約や特性に基づいた実装評価

**注**: AGENTS.mdは英語で記述されていますが、レビューは日本語で実施してください。

## レビュー出力フォーマット

各レビューコメントには以下を含めてください：

1. **優先度**: [Critical/High/Medium/Optional]
2. **カテゴリ**:
   - セキュリティ（認証/認可/XSS/CSRF/データ保護）
   - アーキテクチャ（ディレクトリ構成/関心の分離/依存関係）
   - 可読性（命名/コメント/コード構造）
   - 型安全性（TypeScript型定義/型推論）
   - パフォーマンス（レンダリング/バンドルサイズ/データフェッチ）
   - エラーハンドリング（例外処理/バリデーション/ユーザーフィードバック）
   - テスタビリティ（テスト容易性/モック可能性）
   - データモデル（InstantDBスキーマ設計）
3. **問題点**: 具体的な指摘
4. **理由**: なぜ問題なのか
5. **改善案**: 具体的なコード例を含む提案
6. **参考資料**: 関連するドキュメントやベストプラクティス（あれば）

**重要**: レビューをする上で以下の場合は、改善案の前に明示的に質問してください：

- コードの意図や目的が不明確な場合
- 複数の実装方針が考えられ、判断に追加情報が必要な場合
- プロジェクト固有の規約や方針が不明な場合

## レビュー対象外

以下は意図的な学習目的のため、レビュー対象外とします：

- 過度なアニメーション実装
- 実用性よりも実験性を優先した機能
